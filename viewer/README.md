# Slice Graph Viewer

This directory contains a self-contained static viewer for slice graph JSON generated by `dataflow-engine slice`.

## Serve locally

From the worktree root:

```bash
python3 -m http.server 4173 --directory viewer
```

Open `http://127.0.0.1:4173/index.html`.

Use a local HTTP server instead of `file://` because the viewer uses ES modules and fetches demo fixtures.

## Load graph JSON

- Use `Load Demo` to open the bundled static sample.
- Use the file picker to load a saved `.json` graph.
- Or paste JSON from the CLI into the editor and choose `Load Graph`.

Example fixture generation commands from the repository root:

```bash
cargo run -- slice --static --sv demo/trace_coverage_demo/design.sv --sv demo/trace_coverage_demo/tb.sv --signal TOP.tb.result > viewer/examples/demo-static-slice.json
cargo run -- slice --sv demo/trace_coverage_demo/design.sv --sv demo/trace_coverage_demo/tb.sv --vcd demo/trace_coverage_demo/logs/sim.vcd --signal TOP.tb.result --time 40 --min-time 0 > viewer/examples/demo-dynamic-slice.json
```

## Read the viewer

- Scope rectangles: dashed boxes group blocks by exact HDL scope such as `TOP.tb` or `TOP.tb.dut`.
- Edge labels: text on an edge comes from `edges[*].signal.name`, so it shows the propagated signal.
- Highlight state: the amber block marks the first detected direct driver of the requested target signal; the blue state is the current keyboard or click selection.
- Sidebar details: selecting a block fills in scope, block type, block id, source metadata, and the best snippet currently available in the slice JSON.

Static graphs omit node time annotations. Dynamic graphs keep any per-node `time` values in the loaded viewer state even when the block card labels stay compact.

## Manual checklist

- keyboard focus is visible on controls, the skip link, and rendered block cards in browsers that expose focusable SVG groups
- the sidebar drops below the graph on narrow screens and stays readable on laptop widths
- the page remains usable at roughly 375px, 768px, and 1440px wide; on narrow screens the graph stage scrolls instead of clipping
- reduced-motion preferences remove transition-heavy behavior
- both generated demo fixtures load through the file picker without console errors
